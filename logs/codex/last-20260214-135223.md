**套利玩法/交易策略/风控/常见坑/合规与道德边界**
- 套利玩法：既有 Polymarket 与 Kalshi 之间的合成套利（买 YES、做另一端 NO），也有多事件组合套保（如 Harris vs. GOP spread），都抓住了价格偏离 1 的短暂窗口。 (Round 1 Source 6) (Round 1 Source 10)
- 交易策略：平衡/不平衡市况下分别限制仓位、合成现金及叠加 Maker Rebate，是运行多市场策略的基础；同时需要依赖数据驱动的组合套利逻辑。 (Round 1 Source 2) (Round 1 Source 7)
- 风控：做市商指南和 Maker Rebate 说明了 API + 六大收益策略背后的风险管理流程，实盘中要设置流动性/方向限制并结合 rebated 收益。 (Round 1 Source 7) (Round 1 Source 9)
- 常见坑：套利窗口普遍会被快速抢平，竞争者多、成交迅速收敛，若不及时清仓可能出现两边同时亏损。 (Round 1 Source 3)
- 合规与道德边界：Polymarket 的 Dutch Book 式 $1 结算保障是公平定价的锚点，应把套利视作捕捉短期失配而非操纵市场；跨平台套利也需尊重各平台规则。 (Round 1 Source 11)

**本轮摘要（Telegram）**
- Polymarket 建在 Polygon、借助链下撮合 + 链上结算和 Gnosis CTF，天然存在可推导的二元/负风险套利不等式。 (Round 1 Source 1)
- 有团队靠这些确定性机会悄悄拿下每月 1 万、5 万甚至 20 万以上美元。 (Round 1 Source 1)
- 平台依托 UMA + Polygon，YES/NO 股份价格限定 0～1，成功套利者通过数据驱动的多市场组合寻找 alpha。 (Round 1 Source 2)
- YES + NO 不等于 1 的瞬间就形成价差耕作机会，该类策略风险 <5%、年化 10%～20%。 (Round 1 Source 5)
- Kalshi 与 Polymarket 价格分歧时在一边买 YES、另一边买 NO，可实现跨平台合成套利。 (Round 1 Source 6)
- NO 端流动性和政治事件贡献了可观利润，而体育等场景尚未完全开发。 (Round 1 Source 8)
- 只有 YES 侧流动性时需限仓，双侧流动性时等同持有合成现金，可放量并叠加 Maker Rebate。 (Round 1 Source 7)
- Maker Rebate、做市商 API + 六大盈利策略帮助构建有规则的套利/做市系统。 (Round 1 Source 9)
- 多事件套保（如 Harris vs. GOP spread）在总价低于 1 时锁定差值收益，理论上年化可达 ~40%。 (Round 1 Source 10)
- FKPolyTools 提供 scanMarkets、实时监控、跟单等功能，可自动化跟踪套利与“聪明钱”动向。 (Round 1 Source 4)
- YES+NO 组合被 $1 赎回机制钉死，套利仅来自短暂偏离，守住这个公平锚点即可避免违背平台原则。 (Round 1 Source 11)
- 套利窗口收敛极快，多人抢、价格一起动可能导致双边亏损，因此务必提前设定出场与仓位约束。 (Round 1 Source 3)
