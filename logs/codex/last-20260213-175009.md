**TL;DR**
- OpenClaw/Clawdbot 需要以“安全优先”构建，防止把生活钥匙交给 agent。
- 专用机器 + Tailscale（关闭公网端口）+ 命令白名单是基础防线。
- 配置只读令牌与单向数据流，能显著压缩滥用风险。

**Practical Tips**
- 为 OpenClaw 准备隔离的专用机器，不在日常工作盘上混用，这样更容易评估安全性。(Source 1)
- 通过 Tailscale 等零信任隧道接入，绝不在公网直接打开 OpenClaw 端口，网络面更窄更安全。(Source 1)
- 只允许运行白名单中的命令，拒绝任何动态加载或未经审核的脚本调用。(Source 1)
- 给 OpenClaw 分配到数据源的访问令牌时设置为只读，避免 agent 获得写权限或删除能力。(Source 1)
- 设计处理流程时保证数据只能单向流动，防止敏感输出被 agent 反向写入其他系统。(Source 1)
- 把整个部署当作一个安全项目，不轻易放松权限和监控，维护“生活钥匙不外泄”的理念。(Source 1)

**Pitfalls**
- 不要在普通工作站或多用途服务器上直接部署 OpenClaw，这会把生活钥匙和隐私暴露出去。(Source 1)
- 忽略 Tailscale 等无公网端口保护会让 agent 的访问面暴露在外部扫描下。(Source 1)
- 允许非白名单命令运行时，等于打开了 agent 误操作或被攻击者滥用的新入口。(Source 1)
- 赋予写权限或启用双向数据流容易导致数据泄露、回写或不受控的自动化后果。(Source 1)

**Recommendation**
依照 Jordan Lyall 那篇“生活钥匙不外泄”的翻译指南一步步落实专用机、Tailscale、命令白名单与只读令牌的组合，同时借助 OpenClaw 官方“入门指南”补全对部署顺序与配置界面的理解。(Source 1)(Source 4)
